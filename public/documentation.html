Documentation

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL HTTP Method Request body data format Response body data format
Return a List of all movies to the user /movies GET None A JSON object returning all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:title GET None A JSON object returning the searched Movie
Return data about a genre (description) by name/title (e.g., “Thriller”) movies/genre/:genre GET None A JSON object returning the searched Genre
Return data about a director (bio, birth year, death year) by name /movies/directors/:directorName GET None A JSON object returning the Director information
Allow new users to register /users POST A JSON object holding data about the user to add, structured like: { name : "João", favoriteMovies : [...] } A JSON object holding data about the student that was added, including an ID: { name : "João", favoriteMovies : [...], id : 823fb212 }
Allow users to update their user info (username) /users/:id PUT A JSON object holding data about the info to update, structured like: { name : "Luis", favoriteMovies : [...] } A JSON object holding data about the student updated info, structured like: { name : "Luis", favoriteMovies : [...], }
Allow users to add a movie to their list of favorites /users/:id/:movieTitle POST None A text message indicating that the movie was successfully added to the profile
Allow users to remove a movie from their list of favorites /users/:id/:movieTitle DELETE None A text message indicating that the movie was successfully removed from the profile
Allow existing users to deregister /users/:id DELETE None A text message indicating that the profile was successfully removed
